## How to regenerate

The files are generated by `scripts/generate-qac.mjs`, which runs automatically
before `vite dev` and `vite build` **if the files do not yet exist**.

### Normal regeneration

Delete the existing files, then start the dev server (or run the build):

```bash
rm artifacts/rah-e-safia/public/data/qac-morphology.json \
   artifacts/rah-e-safia/public/data/qac-roots.json
pnpm --filter @workspace/rah-e-safia run generate-qac
```

Or force-regenerate without deleting:

```bash
pnpm --filter @workspace/rah-e-safia run generate-qac -- --force
```

### What the script does

1. Downloads the QAC morphology archive from `corpus.quran.com/download/`.
2. Extracts `quranic-corpus-morphology-0.4.txt` from the zip.
3. Parses every STEM token and converts Buckwalter transliteration to Arabic Unicode.
4. Writes `qac-morphology.json` and `qac-roots.json`.
5. Prints word count, root count, and file sizes — **if the numbers look wrong
   compared to the table above, the upstream data may have changed**.

### Verifying the committed files

Run the integrity check at any time:

```bash
pnpm --filter @workspace/rah-e-safia run verify-qac
```

This exits with code 0 if both files are present and meet the minimum key counts,
or exits with code 1 and a descriptive error message if something is wrong.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| Word Study shows "loading" indefinitely | Missing or empty JSON files | Regenerate (see above) |
| Record counts are much lower than expected | Upstream format changed | Check corpus.quran.com for changes; update the parser |
| Download fails with HTTP error | Corpus site unavailable | Retry later or obtain the archive manually |
| `unzip` not found | Missing system dependency | Install `unzip` (`apt install unzip` / `nix-env -i unzip`) |
